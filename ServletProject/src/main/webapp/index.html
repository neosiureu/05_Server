서버 제출 테스트1

이름, 나이를 입력받은 후 서버(자바코드)로 제출하겠다

※ 준비물 ※

1) form태그가 필요 (내부에 작성한 input 값을 지정된 경로로 제출)
  action method
  1-2) action 속성에 서버 경로 넣음
  1-3) method 속성에 get post



제출 = 클라이언트가 서버로 data를 가지고 request한다는 뜻



2) input 태그: 값을 입력하는 양식 제공
-name (*중요*): 서버로 제출할 때 key역할
-value 또는 입력 값: 서버로 제출할 때 value 역할을 함




 GET: GET을 통한 요청은 URL주소 끝에 파라미터로 포함되며 전송되며 이 부분의 부분을 쿼리 스트링이라 부른다.


 방식: URL 끝에 문자 ? 를 붙인 후 , 쌍1&쌍2&쌍3... (쌍은 변수명 = 값) 
  ex) www.ex.com/?name1=value1&name2=value2... 로 주소가 넘어감 


  이 때 서버에서는 name1과 name2라는 파라미터 이름으로 각각 값인 value1과 value2값을 전달받을 수 있다



 단순함 ＋  웹 캐싱 가능 (주소저장=북마크, 즐겨찾기, 주소 공유) 


 url뒤를 보면 이름과 값이 다 나오기에 보안에 취약하고, 요청 데이터의 길이 제한 존재
